XML - Extensible Markup Language

2

Allgemeines

  • 1996 von einer Arbeitsgruppe des W3C entwickelt
  • ist ein freier Standard
  • Einsatzgebiete:
  • Speicherformat in Applikationen
  • XML als Format um Informationen auszutauschen

3

XML – Dokumente (1)

  • Prolog und Dokumenttyp-Deklaration
  • Versionsnummer,
  • externe oder interne Dokumenttyp-Deklaration, wird festgelegt

<?xml version="1.0"?>

<!DOCTYPE gruss SYSTEM "hallo.dtd">

<gruss>Hallo Welt!</gruss>

4

XML – Dokumente (2)

  • „wellformed“

entspricht einem syntaktisch korrekten XML-Dokument.zB.

  • die XML-Version sollte deklariert sein
  • Jedes Start-Tag muss mit einem End-Tag abgeschlossen werden
  • ein Attribut darf nicht zweimal im gleichen Tag vorkommen

5

XML – Dokumente (3)

  • „valid“ (gültig)

das Dokument besitzt eine DTD und hält sich an die darin formulierten Beschränkungen

6

XML – Dokumente (4)

  • Dokument Typ Declaration (DTD)
  • Enthält Markup Deklarationen
  • Ist eine Reihe von Element-, Attributs- und Entity-Deklarationen
  • Legt die Syntax eines XML-Dokumentes fest

7

Logische Strukturen (1)

  • Elemente

<!ELEMENT Elementname Content-specification>

  • Elementname = Elementtyp
  • Müssen mit einem Buchstaben oder einem Unterstrich beginnen
  • Dürfen nicht mit den Buchstaben XML beginnen

8

Logische Strukturen (1)

  • Content Specification

<!ELEMENT Elementname Content-specification>

Dabei unterscheidet man:

  • Element Content
  • Mixed Content
  • ANY
  • EMPTY

9

Beispiel Element

<!ELEMENT Kapitel (KapTitel, (Para | Zwischentit)+) >

Erklärung:

<!ELEMENT = es folgt eine element declaration

Kapitel = Name des Elementes

( ) = Unterelemente

, = darauf folgend

| = oder

+ = eines oder mehrere [2]

10

Logische Strukturen (2)

  • Attribute

<!ELEMENT Elementname (Inhalt)>

<!ATTLIST Elementname

Attributname_1 Attributtype [#REQUIRED|#IMPLIED|#FIXED "Wert"|Defaultwert]

>

11

Logische Strukturen (3)

  • Attributtyp

Dabei unterscheidet man:

  • CDATA
  • ENTITY
  • ENTITIES
  • ID
  • IDREF / IDREFS
  • NMTOKEN / NMTOKENS

12

Logische Strukturen (4)

  • Entities

kann folgende Funktionen haben:

  • Ersetzung eines Strings durch einen anderen String
  • Verknüpfung mit einem anderen XML-File
  • Platzhalter für Graphik oder andere Nicht-XML-Daten, die eingefügt werden

13

Logische Strukturen (5)

<!ENTITY 5BDT „PRE, PR, ADAT">

Unterteilt in:

  • General Entities
  • Parsed
  • Internal
  • external
  • Unparsed
  • Parameter Entities

durch %entity_name in der DTD verwendet

14

Parser (1)

Lassen sich unterscheiden:

  • Validierende Parser

überprüft ob das XML seiner DTD gehorcht (valid)

  • Nicht-Validierende Parser

überprüft ob es der XML Syntax folgt (wellformed)

15

Parser (2)

  • SAX
  • arbeitet "event based„
  • Ereignisse können durch Eventhandler behandelt werden
  • SAX-API definiert mehrere Eventhandler
  • Document Handler
  • Error Handler

16

Parser (3)

  • Vorteile SAX
  • Hohe Geschwindigkeit
  • Große Datenmengen können verarbeitet werden

17

Parser (4)

  • DOM (Document Object Model)
  • Daten werden als Document Object abgespeichert (DOM-Tree)
  • Definiert mehrere Interfaces

-> Node ist das wichtigste, davon sind alle anderen abgeleitet
